# Scrim — native shell (Expo / React Native)

The iPhone app. It wraps the live web app (`https://scrim-backend-wxcn.onrender.com`)
in a native shell, so every push to `main` updates the app instantly — no App
Store release needed. Native capabilities (LiDAR depth, ARKit scan capture,
golden-hour push notifications) get added here as Expo modules and talk to the
page through `window.__SCRIM_NATIVE__` and `postMessage`.

## Run it on your Mac

One-time setup:

1. Open Xcode once, accept the license, and let it install the iOS components
   it offers on first launch.
2. Install Node.js (LTS) from https://nodejs.org if you don't have it.
3. Install CocoaPods: `brew install cocoapods` (or `sudo gem install cocoapods`).

Then:

```bash
git clone https://github.com/akehar/SCRIM.git   # or git pull if you have it
cd SCRIM/native
npm install
npx expo run:ios          # builds and opens the iOS Simulator
```

The first build takes a few minutes (it generates the `ios/` project and
installs pods). After that it's fast.

## Run it on your actual iPhone

The Simulator has no camera, GPS, or motion sensors — the phone is where Scrim
makes sense.

1. Plug in your iPhone with a cable and trust the computer when it asks.
2. Run `npx expo run:ios --device` and pick your phone.
3. First time only, Xcode needs a signing team: open `ios/Scrim.xcworkspace`,
   select the Scrim target → Signing & Capabilities → check "Automatically
   manage signing" and sign in with your Apple ID. A free Apple ID works —
   the app just re-signs every 7 days when run from Xcode. The $99/year Apple
   Developer Program removes that limit and unlocks TestFlight + App Store.
4. On the phone: Settings → General → VPN & Device Management → trust your
   developer certificate. iOS may also ask you to enable Developer Mode
   (Settings → Privacy & Security → Developer Mode → on, then reboot).

## What lives where

- `App.js` — the whole shell: WebView with camera/location permissions wired,
  Field Notes loading and offline states, external links handed to Safari.
- `app.json` — app identity (name, bundle id `com.akehar.scrim`), permission
  strings, icons.
- `assets/` — icons generated from `public/icon.svg` (regenerate with the
  icon script if the emblem changes).
- `ios/` — generated by `expo run:ios`, never committed (`expo prebuild`
  recreates it from `app.json` at any time).
